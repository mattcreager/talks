Make it real-time

Context and events are used to maintain a local representation of your remote data

Event Bindings

  • Set: An existing value has changed
  • Add: A new property has been added
  • Remove: A property has been removed

Context

{
  "command": "SET",
  "targetKey": "roster/aRoStErID",
  "currentKey": "roster/",
  "value": "aNewValue",
  "userId": "user_5",
  "position": {
    "current": 4,
    "previous": 6
  },
  "events": {
    "current": 2,
    "total": 3
  }
}

The Initializing Fetch

  • Primes the model
  • Ensures a valid state
  • Publishes a notification

Return an object now
populate it later

Promises (Q/$q) for the one-hit wonders, events for the other stuff

Persist Local Changes (if you must)

  • Watch the local model
  • Calculate the diff
  • Push changes upstream

Caution: Kiss your error locality, performance and sanity good-bye

Conclusion

Models provide an API for accessing and manipulating data

Data encapsulation is important, especially in Angular

Make a model real-time by:

  • 1. Monitoring remote events
  • 2. Performing an initial fetch
  • 3. Monitoring and persisting local state
